設計思維──愛在大稻埕

  • 作品名稱： 愛在大稻埕
  • 作品說明：
    這是一款以大稻埕的霞海城隍廟做為發想的戀愛旅遊APP。使用者可以透過七個不同的特色關卡來收集紅線的片段，串聯成愛心的圖樣後便能製作愛情御守。而遊戲中出現的關卡地點皆位在大稻埕，內容也和大稻埕的文化與傳統有關。另外，附加擲筊和抽籤的小遊戲和社群功能，使用者可以將製作好的愛情御守掛在愛情樹上，彼此互相留言和寄信，分享幸福。
  • 工作分配：
    除了APP主題和功能的共同討論，我另外負責規劃到大稻埕的訪查路線，以及設計各關卡內容，還有最後企畫書的第四點──遊戲功能與特色。

文化脈絡與故事原型──Makiyo友人毆打運將事件

  • 作品名稱： Makiyo友人毆打運將事件──「演藝圈的黑暗」
  • 作品說明：
    針對這起新聞事件，去搜尋有關事件本身和當事人的背景資訊，經過整理與分析以重新定義當事人在事件中的角色原型，並且從中選出一個最想表達的insight，然後透過包裝、拆解與重組，讓事件以不同的樣貌再現(represent)於觀眾眼前。
    在Makiyo事件中，我們的選出的insight是「演藝圈的黑暗」，試圖表達一旦進入演藝圈，藝人的形象只能被螢光幕所操控，何謂真相並不重要，或許Makiyo等人已經深深悔改，但是為了收視率或者迎合大眾的口味，藝人只能任人宰割。
  • 工作分配：
    組員一起討論整個事件的故事脈絡，挑選insight和呈現方式。我個人負責分析事件中有趣之處以及Ma媽的角色原型，還有在期末影片中替湘瑩配音，並製作部分道具。

資料庫應用──台灣旅遊交流平台

  • 作品名稱： 台灣旅遊交流平台
  • 作品說明：
    以台灣的旅遊交流平台為主題，設定它應該要包含的功能和服務，包含可以查詢相關旅遊景點、餐廳和住宿資訊以及評價，還有購物車功能讓使用者規劃旅遊行程等。接者將這些描述依序轉換成「實體和屬性」、「實體關係圖」、「關聯模式圖」以及SQL，然後蒐集相關資料和建立表單，最後每個組員各設計兩個題目，再以MySQL做測試。
  • 工作分配：
    除了共同討論主題、功能和服務，其他都是平均分配工作，例如：各畫三個實體。

觸控介面──Tempo Run

  • 作品名稱： Tempo Run
  • 作品說明：
    這是一款幫助慢跑者掌控步頻的APP，特色在於結合電療貼片。使用者藉由將貼片黏貼在頸部或手臂，可以感受到貼片根據APP設定的打點頻率來調整步頻。
  • 工作分配：
    除了共同的工作內容，例如：創意發想、使用者訪談、再檢討，我另外負責營運模式與市場推廣方式的構思，以及擔任參與金驗獎比賽的組長：負責和主辦單位聯絡，及時公告消息給組員，還有統整企畫書和PTT等。

Web程式設計──失物招領

  • 作品名稱： 失物招領
  • 作品說明：
    有鑑於警政署、捷運和客運等失物招領網站的設計不良，我們以使用者的角度來重新設計一個介面更直觀、更有效率的遺失物搜尋網站。功能方面，除了一般搜尋之外，為了有尋路回頭找遺失物習慣的使用者，還新增了地圖搜尋功能。
  • 工作分配：
    除了一起討論目標與功能和使用者需求的問卷之外，包含我每個人設計其中一頁介面的設計，我也負責統整整個網站的介面設計，以及寫CSS的部分。
